
## 什么是“拓展段”
在芯片里，GPIO 配置寄存器不是一整块连续表覆盖所有 GPIO，而是分成两段（两张表）：

- **基础段**：`core_gpio_conf + GPIO号`  
  适用于较小编号的一批 GPIO（通常是 0~15）
- **拓展段**：`core_gpio_conf_hi + (GPIO号-16)`  
  适用于更高编号的 GPIO（通常是 16 及以上）

工程里能看到两个基址寄存器定义：
- `core_gpio_conf`（见 `D:\YiChip_workspace\Stack\format\regs.format:52`）
- `core_gpio_conf_hi`（见 `D:\YiChip_workspace\Stack\format\regs.format:105`）

并且在 `D:\YiChip_workspace\Stack\program\peripherals.prog:2183` 附近，`gpio_get_config`/相关逻辑明确写了：当 GPIO 号超过某个阈值时，会切换到 `core_gpio_conf_hi`，并对 GPIO 号做 `-16` 的偏移。

所以我这里说的“拓展段”，就是指 **GPIO16 及以上那一段要用 `core_gpio_conf_hi` 来配置**。

---

## 为什么用 `gpio_config_function_int` 就能正常发送
你之前用的是这种写法（问题点在 GPIO16）：
- `jam gpcfg_uart_txd, core_gpio_conf+16`

如果 GPIO16 实际属于“拓展段”，那么：
- `core_gpio_conf+16` **并不是 GPIO16 的配置寄存器**（写错地方了）
- 结果是：GPIO16 并没有真正被切到 `UART_TXD` 功能上，芯片当然就发不出数据到你接的 TX 线上

而 `gpio_config_function_int` 的作用是：
- 它会根据你传入的 GPIO 号（15 或 16），**自动选择**写 `core_gpio_conf` 还是 `core_gpio_conf_hi`
- 对于 GPIO16，它会走“拓展段”路径，做 `GPIO-16` 偏移后写到正确的配置寄存器
- 因此 GPIO16 真正变成了 UART_TXD，串口助手就能收到数据了

一句话总结：
- 你之前“UART 本身初始化没问题”，但 **TX 引脚复用写错寄存器**；`gpio_config_function_int` 帮你把“写到哪张 GPIO 配置表”这件事做对了，所以立刻就通了。

## Ring Buffer 是什么、怎么初始化、怎么收发

### 1 Ring Buffer 定义
- TX/RX 各一段连续 RAM，硬件 DMA 自动维护读写指针，指针到尾部会回环。
- 软件要做的核心动作：
  - TX：写入 ring → 提交写指针（写回 `core_uart_twptr`）
  - RX：读出 ring → 提交读指针（写回 `core_uart_rrptr`）

### 2 初始化要求（4 个地址打包为 8 字节）
`uarta_init_dma_mem` 需要按顺序提供：
1) RX start（16bit）
2) RX end（16bit）
3) TX start（16bit）
4) TX end（16bit）

## 实际落地：GPIO15/16 UART 初始化、发送、接收

### 1 创建内存布局（format）
- 定义：
  - `mem_uart_rx_ringbuf` / `mem_uart_tx_ringbuf`（各 16 字节 ring）
  - `mem_pdatatemp`（8 字节：打包 4 个地址用于 `uarta_init_dma_mem`）

### 2 UART 初始化函数（修复后的关键点）
- `my_feature_uart_init`：
  - GPIO 复用：改为使用 `gpio_config_function_int` 配 GPIO15/16（避免 GPIO16 扩展段写错表）
  - ring buffer：将 `mem_uart_rx_ringbuf` / `mem_uart_tx_ringbuf` 的 start/end 打包到 `mem_pdatatemp` 后调用 `uarta_init_dma_mem`
  - 波特率：设为 115200（`uarta_calc_baud_rate_config`）
  - 使能 UART：写 `core_uart_ctrl`

### 3 发送 0x1F 的实现
- 思路：
  - 将 `0x1f` 写入 1 字节临时变量
  - `uarta_prepare_tx` → `uart_copy_tx_bytes` → `uarta_send`

# 串口初始化代码
```
my_feature_uart_init:

    // 1) 配 GPIO 复用

    arg gpcfg_uart_txd,temp

    setarg My_UART_TX_NUM         //tx引脚

    call gpio_config_function_int

    arg gpcfg_uart_rxd|gpcfg_pullup,temp

    setarg My_UART_RX_NUM         //rx引脚

    call gpio_config_function_int

    // 2) 关闭 UART

    fetch 1,core_uart_ctrl

    set0 BIT_UART_CONTROL_ENABLE,pdata

    store 1,core_uart_ctrl

    // 3) 打包 RX/TX ring buffer 地址到 mem_pdatatemp

    setarg mem_uart_rx_ringbuf

    store 2,mem_pdatatemp

    setarg mem_uart_rx_ringbuf+15      //需要配置ringbuf长度，这里是16字节的ringbuf

    istore 2,contw

    setarg mem_uart_tx_ringbuf

    istore 2,contw

    setarg mem_uart_tx_ringbuf+15      //同上

    istore 2,contw

    // 4) 初始化 UARTA DMA ring buffer

    fetch 8,mem_pdatatemp

    call uarta_init_dma_mem

    // 5) 时钟 + 波特率

    call uart_clock_select_main_freq_crystal

    arg 115200,temp                    //配置波特率

    call uarta_calc_baud_rate_config

    // 6) 使能 UART

    arg FUN_UART_CONTROL_ENABLE|FUN_UART_CONTROL_SETTING_RATE_BAUD,temp

    storet 1,core_uart_ctrl

    rtn
```

# 串口接收数据代码

core_uart_rxitems中存有当前未读取的数据的个数
```
my_uart_receive_bytes:

    fetch 2,core_uart_rxitems    // pdata = 当前ringbuf中字节数

    rtn blank                    // 没有数据直接返回

    store 2 , mem_rx_message_sum           // loopcnt = 要处理的字节数

my_uart_receive_bytes_loop:

    call uarta_prepare_rx        // 准备rx指针

    // 从ring中读1字节到临时位置

    arg mem_pdata,contw      // 用mem_pdata存单字节

    arg 1,loopcnt                //通过设置loopcnt来选择读取多少字节的数据

    call uart_copy_rx_bytes      // 读1字节 → mem_pdata

    call uarta_rxdone            // 更新ring读指针

    fetch 1,mem_pdata        // pdata = 刚读到的字节

    fetch 2,mem_rx_message_sum

    increase -1,pdata

    store 2,mem_rx_message_sum

    nbranch my_uart_receive_bytes_loop, blank  // 继续处理剩余字节

    rtn
```

# 串口发送数据

```
my_uart_send_bytes:
    call uarta_prepare_tx

    arg mem_tx_buf,contr             // 源：mem_tx_buf为要发送的数据的头指针

    fetch 1,mem_tx_len              // mem_tx_len为要发送的数据的字节数

    copy pdata,loopcnt               

    call uart_copy_tx_bytes

    call uarta_send

    rtn
```
