# 双仓库同步配置说明

## 仓库结构

```
obsidian_notes/
├── .git/              # Gitee 私人仓库
├── .gitmodules        # submodule 配置
├── 技术笔记/          # GitHub 公开仓库 (submodule)
│   ├── .git/
│   └── *.md          # 技术类笔记
├── 日记/              # 私人笔记
├── *.md              # 其他私人笔记
└── git 嵌套解除.md
```

## 仓库关联

| 目录 | 远程仓库 | 可见性 |
|------|----------|--------|
| 根目录 | `git@gitee.com:wusj1001/obsidian_notes.git` | 私人 |
| 技术笔记/ | `git@github.com:WuSJ1001/embedded-notes.git` | 公开 |

---

## 工作流程

### 修改技术笔记（同步到 GitHub）

```bash
cd 技术笔记
git add .
git commit -m "更新技术笔记"
git push origin main
```

### 修改私人笔记（同步到 Gitee）

```bash
# 回到主目录
cd ..

git add .
git commit -m "更新私人笔记"
git push origin main
```

### 更新技术笔记 submodule 引用

如果 技术笔记 有新的 commit，需要在主仓库中更新引用：

```bash
git add 技术笔记
git commit -m "更新技术笔记 submodule 引用"
git push origin main
```

---

## Obsidian 使用方式

1. 打开 Obsidian，将 **整个 `obsidian_notes` 目录** 添加为仓库
2. 所有笔记（包括 技术笔记 中的）都可以正常编辑
3. 根据上述工作流程分别推送更改

---

## 快速脚本

### 同步所有更改到两个仓库

```bash
# Windows (bat 脚本)
@echo off
cd /d "%~dp0"

echo === 同步技术笔记到 GitHub ===
cd 技术笔记
git add .
git commit -m "auto: vault backup"
git push origin main

cd ..

echo === 同步私人笔记到 Gitee ===
git add .
git commit -m "auto: vault backup"
git push origin main

echo === 同步完成 ===
```

### 仅检查状态

```bash
# 查看主仓库状态
git status

# 查看技术笔记状态
cd 技术笔记 && git status
```

---

## 注意事项

1. **技术笔记** 中的文件变更只会推送到 GitHub，不会直接同步到 Gitee
2. Gitee 仓库只记录 技术笔记 submodule 的 **commit 引用**，不包含实际文件内容
3. 拉取他人更改时，需要分别拉取两个仓库：
   ```bash
   # 拉取主仓库
   git pull origin main

   # 拉取技术笔记 submodule
   git submodule update --init --recursive
   ```
