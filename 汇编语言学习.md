# 最重要的两个寄存器：`pdata`和`temp`

汇编指令中有很多指令都是将一些函数返回的值或者寄存器中的值读出暂存入`pdata`寄存器中，`blank`是一个标志位，该标志位通过该寄存器中的值来进行判断。`temp`是一个通用寄存器，在对数据的操作使用一个寄存器不足以满足时，可以使用`temp`来对数据进行存储。（想象成左右手）
另外还有一个64位的`regab`寄存器该寄存器被分为了`rega`和`regb`寄存器，通常可以用来存储地址指针
同时在使用寄存器时还有一些额外的特定字节长度的寄存器（`regc，alarm`等）可以在有需要时使用，具体可以到手册中查询
在需要保存多个配置参数时，还有`regxt`共12个寄存器可以使用通过索引`regxt_index`来访问

# 运算指令和逻辑运算

`and，sub，add，or，xor`运算后有三个参数，分别是`regr`源寄存器`immediate`立即数`regw`目标寄存器，带i前缀的通常就是和`pdata`寄存器相关的操作

# 寻址指令

`jam`和`hjam`负责来存立即数
`fetch`和`fetcht`负责来读取立即数，`fetch`作用是将立即数读取到`padata`中，`fetcht`则是将数据读取到`temp`中，带`i`前缀的就是间接寻址，在间接寻址中，函数会根据第二个变量中存储的地址去寻址。
`store`和`storet`作用是将寄存器中的数据写入内存中
# 条件语句

条件语句的结构：
<判断指令><条件>
如果条件满足执行该指令，不满足就忽略
在条件中一般使用标志位来判断，其中标志位`blank`的数值由`pdata`中是否有数据来决定，没有数值或为零时置1，否则是0（有点反逻辑）


**思考：在已经有了call指令的情况下，为什么还要有一个branch指令，branch指令不能rtn的意义是什么？**
### 1. 核心区别：是否保存“回头路”

- **call 指令：** 在跳转到目标地址之前，CPU 会自动把**下一条指令的地址**（即返回地址）保存到一个特殊的地方（通常是堆栈或特定的链接寄存器）。执行 rtn 时，CPU 从那个地方取出地址并跳回来。
    
- **branch 指令：** 仅仅是修改 PC（程序计数器）的值。它**不保存**任何返回地址，就像直接把人传送到了另一个地方，但不留任何路标。
    
### 2. 为什么需要 branch？（即便有了 call）

如果在所有的跳转中都使用 call，会导致以下问题：

#### A. 堆栈溢出（Stack Overflow）

这是最关键的原因。蓝牙芯片（如 YC1021）内部的资源（RAM/寄存器）非常有限。call 存储返回地址是需要占用空间的。

- 如果你只是想实现一个 **If-Else** 判断或者 **Loop** 循环，却使用了 call，那么每判断一次、每循环一次，堆栈里就会多存一个地址。
    
- 如果不回头（不执行 rtn），堆栈很快就会被填满，导致芯片崩溃。
    
- **branch 的意义：** 它是“一次性”的跳转，不占用堆栈空间，适合处理无限循环或长距离逻辑切换。
    
#### B. 构建基础逻辑（If/While/Switch）

汇编里的 branch 是为了构建基础控制流。

- **实现循环：** 手册第 50 页的 usb_tx_loop 示例中，最后一行是 loop usb_tx_loop（这其实是一种带计数的 branch）。如果没有这种不回头的跳转，你无法构建“跳回开头重新开始”的结构。
    
- **If-Else 结构：** 当条件不满足时，我们需要跳过一段代码。这种“跳过”操作是不需要回头的，用 branch 最为高效。
